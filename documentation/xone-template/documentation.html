Introduction

You bought an HTML / CSS template, so you need some basic knowledge for those 2 languages. The Best way is to play around with the different elements/sections but you find some helpfull tips and explanations how this template is build up and works.
Please compare all the explanation from this file with the original files to have a better understanding.

General Rules

These are general classes, id's or tags which you will find several times in almost all html files.

#page-content: you will find this element on every page. It contains all main content (header,maincontent,footer)

.wrapper: it wrapps the element to a the main content width. (1100px = default / 940px for smaller screens / 740px for tablet (portrait) / 300px for phones).

.main-content: if you use a layout with a sidebar (aside) you need add this classname to the main content. Use this tag in combination with right-float or left-float

aside: this is the sidebar. Use this tag in combination with right-float or left-float.

.left-float: use this classname for any element you want to float to the left.

.right-float: use this classname for any element you want to float to the right.

.clearfix: for all tags which contains any floating element.

.align-center: use this class if you want to align the text from a whole section or just an element.

.text-light: this class is needed if you use a darker background for a section (for example for the footer or a dark parallay section)

EXAMPLE OF THE ABOVE EXPLANATIONS

...
<div class="wrapper clearfix">

 <div class="main-content left-float">
 </div>

 <aside class="right-float">
 </aside>

</div> 
...

The left / right-float class are very helpfull. In a template with sidebar for example you can easily change the position of the sidebar by just changing the classname without doing any changes at the css.

You can choose custom id names for the different "sections". Just make sure that the same name is used for the navigation and that they are unique (for the index templates with scroll effect).

In the index.html, the contact section is the only section which does this, because it's the only section which have no parallax element. For all other sections there has been added an additional div tag with the wrapper class (See example below).

You also see how to add the section-title.

The sections have a default top padding (no bottom-padding). For all templates I always give the last section an additional spacer in order not to stick to the footer.

Fonts

You have to include the fonts in every html you use/create. Make sure to include them in the head before the css files.

<link href='http://fonts.googleapis.com/css?family=Open+Sans:300,400,800,600|
Raleway:300,600,900' rel='stylesheet' type='text/css'>

You can see that I included 2 Fonts (Open Sans & Raleway) in different weights.
So when I choose this font in the style.css I always added the weight I wanted. 

h1, h2, h3, h4, h5, h6 {
 font-family: 'Raleway'; 
 font-weight: 400;
 color: #262626;
}
 
.section-title h2 { 
 font-weight: 900;  
}

At line 8 of this example above I added the weight 900 for all section-title.

Menu / Navigation

Please use the following markup for the main navigation. 

<nav id="main-nav">
   <ul>
      <li class="current-menu-item"><a href="#home">Home</a><li>
      <li><a href="#about" class="scroll-to">About</a>
	  ...
      <li><a href="elements.html">Elements</a>
         <ul class="sub-menu">
            <li><a href="elements.html">General Elements</li>
            ...
         </ul>   
      <li>
   </ul>
</nav>

class="current-menu-item": add this classname to the actual li element. In the elements.html for example it would be added to this li.

href="#about": this must have the same name then the id of the section you want to assign starting by a hashtag.

class="scroll-to": add this class to all menu items which should scroll to an section. You can also use this class on any anchor tag in combination with the right href value.
You see that "Elements" does not this classname because it's linking to another html page.

Important: Depending on your logo height you need to adapt this value in the css. Please see "Changing Logo" for detailed info.



Bullet navigation

As you probably have seen the default/text navigation transforms into a bullet navigation after scrolling down. Place the following markup just after the default navigation. 

<nav id="menu-controls">
 <ul>
  <li class="current-menu-item"><a href="#home" class="scroll-to">
   <span class="c-dot"></span>Home</span></a></li>
  <li><a href="#about" class="scroll-to">
   <span class="c-dot"></span><span class="c-name">About</span></a></li>
  ...
 </ul>
</nav>

Just delete this navigation markup on the files if you don't want it.

Important: Do not place any submenu for the bullet navigation.

Filter

The filter is part of the isotope script. It will only work on masonry-items.
All filter anchors needs a data-option-value attribute with the correct filter name (example: .branding) 

<ul id="portfolio-filter" class="filter clearfix" data-related-grid="portfolio-grid">
  <li><a class="active" href="#" data-option-value="*">All</a></li>
  <li><a data-option-value=".web" href="#" title="Web">Web</a></li>
  <li><a data-option-value=".branding" href="#" title="Motion">Motion</a></li>
  <li><a data-option-value=".motion" href="#" title="Print">Print</a></li>
</ul>

It's important that the value of data-related-grid is the same than the id name of the grid (see above).

Background Video

In fact you can add a background video to every kind of element but I usually created an "horizontalsection" to do so. Make sure that this element is not wrapped in a .wrapper element in order to make it fullwidth.

Make sure to provide your video in 3 different formats:

  • mp4
  • webm
  • ogg

Add the classname "videobg-section" (and optionally 'text-light' if you use a dark video) to the element, and fill in the different "data-" options. 

<div id="parallax-video" class="horizontalsection text-light videobg-section"
 data-videofile="files/uploads/steven" 
 data-videowidth="1280"
 data-videoheight="720"
 data-videoposter="files/uploads/poster-image.jpg"
 data-videoparallax="true"
 data-videooverlaycolor="#000000"
 data-videooverlayopacity="0.5"
 data-sound="true">
  • data-videofile -> please add the path to your files without the file ending (ie: .mp4)
  • data-videowidth + data-videoheight -> very important that you enter the right values
  • data-videoposter -> this poster is shown for devices which don't support background videos
  • data-videoparallax -> true or false for a parallax effect
  • data-videooverlaycolor -> optional overlay color / leave empty if not wanted
  • data-videooverlayopacity -> opacity value for the overlay
  • data-sound -> if true, the video also plays the sound with option to mute it manually

Parallax Backgrounds

Same as background video, you could add a parallax image to every element but I usually created an "horizontalsection" to do so. Make sure that this element is not wrapped in a .wrapper element in order to make it fullwidth.

Add the classname "parallax-section" (and optionally 'text-light' if you use a dark background) to the element. 

<div id="facts" class="horizontalsection text-light parallax-section">

Once done, you just need to add your background image to this section via css.
Please make sure to use a center center alignment and a cover background size. This is important for the devices which does not support a smooth parallax effect. 

#facts.parallax-section { 
 background:url(../uploads/parallax-bg1.jpg) center center repeat; background-size: cover; 
}

Google Map

Search first for your longitude and latitude on itouchmap.com.
Once you got them use this script to add your google map. The map will always take the full width of the parent container. You just need to set up the height vie inline css.

<div id="map" class="header-map" style="height:400px;"></div>
<script type="text/javascript" src="http://maps.google.com/maps/api/js?sensor=false"></script>
<script type="text/javascript">
   function mapinitialize() {
      var latlng = new google.maps.LatLng(-33.86938,151.204834);
      var myOptions = {
         zoom: 14,
         center: latlng
         scrollwheel: true,
         scaleControl: false,
         disableDefaultUI: false,
         mapTypeId: google.maps.MapTypeId.ROADMAP
      };
      
      var map = new google.maps.Map(document.getElementById("map"),myOptions);
      var image = "files/uploads/map-pin.png";
      var marker = new google.maps.Marker({
         map: map,
         icon: image,
         position: map.getCenter()
      });
      
      var contentString = '<b>Office</b><br>Streetname 13<br>50000 Sydney';
      var infowindow = new google.maps.InfoWindow({
         content: contentString
      });
      
      google.maps.event.addListener(marker, 'click', function() {
         infowindow.open(map,marker);
      });
      
   }
   mapinitialize();
</script>

Change the map-pin image if you want and the Text for the infobox.

Spacer

If you already took a look at the html files you probably met some "spacer" elements. These elements have been created to give you control over some spacings.
For example I used them to add a spacing at the bottom of the last section of the page.

Just add a div with the classname spacer + the height of the spacer 

<div class="spacer spacer-big"></div>
  • spacer-big = 100px
  • spacer-medium = 60px
  • spacer-small = 40px
  • spacer-mini = 15px


Seperator

The seperator will create a seperator line.
It's used for the section title for example, but you can use it where you want 

<div class="seperator size-small"><span></span></div>

By default it has a default width of 100% and a height of 2px but you can change it by just adding an additional classname with the size

  • size-medium = width of 220px
  • size-small = width of 100px
  • size-mini = width of 40px
  • height-small = height of 1px


Horizontal Section

The horizontal section is used for make use of different background (parallax,videobg or bg color). 

<div class="horizontalsection">
 <div class="horizontalinner wrapper">
  ...
 </div>
</div>

Use the markup above. You see that the horizontalsection element is always followed by a horizontalinner. This is important. The horizontalsection has nor margin-top or margin-bottom because you could make use of another horizontalsection just below another and a margin wouldn't look good. If you want a spacing between section, use a spacer element.



Animations

In general you can use the animation for any element, but I really recommend to use it for normal/small elements like columns,icons,images, etc. I would'tn use it for whole sections.

Add the classname sr-animation to the wanted element, the animation effect and the optional delay value. 

 <div class="column one-third sr-animation sr-animation-zoomin" data-delay="200">
  • sr-animation-fromtop
  • sr-animation-frombottom
  • sr-animation-fromleft
  • sr-animation-fromright
  • sr-animation-fade
  • sr-animation-zoomin
  • sr-animation-zoomout

Retina



Css (Template Icons)

This template is Retina Ready which means that all icons (files/images) which are part of the template will be displayed on high-resolution on retina devices.
This is managed in the retina.css. Please take a look. If you add new background-images to the css please add them also in the retina.css.



Javascript (Your custom images)

All pages include the retina.js. When your users load a page, retina.js checks each image on the page to see if there is a high-resolution (@2x) version of that image on your server. If a high-resolution variant exists, the script will swap in that image in-place.

Example
The logo image file is available in 2 version (xone-logo.png + xone-logo@2x.png). On retina devices the @2x.png file is included.

Note
The retina.js can only apply on online server and won't work if files are opened locally.

Css Files

You should find the following css files

  1. style.css -> main style !important
  2. font-awesome.min.css -> css for font awsome icons !important
  3. retina.css -> replacing images for retina devices !important
  4. mqueries.css -> css rules for being responsive !important
  5. flexslider.css -> style for flexslider (content sliders)
  6. isotope.css -> style for isotope animations (portfolio filter)
  7. easy-opener.css -> style for easy-opener (lightbox for videos)
  8. dark.css -> style for a dark color sheme (see below)
  9. jplayer/jplayer.css -> style for jplayer plugin (selhosted video/audio)
  10. rs-plugin/css/settings.css -> style for revolution slider plugin (home slider)

All css files with !important must be included in all html files.
The other files are only needed if you use the appropriate jquery script in the file.

Example:
The jplayer.css from the jplayer plugin is only included in the blog.html because that's the only file which uses the plugin.

Note: The mqueries.css must always be the last css file!



Dark

If you want to have a dark color sheme, just include the dark.css before the mqueries.css css files. Now, if you want to change some colors (ie. background for header), please do the changes in the dark.css because this is now the file which manages all dark/light colors except the 'main color' (blue in the original files). 

...
<link rel='stylesheet' href='files/css/retina.css' type='text/css' media='all' />
<link rel='stylesheet' href='files/css/dark.css' type='text/css' media='all' />
<link rel='stylesheet' href='files/css/mqueries.css' type='text/css' media='all' />
<!-- CSS -->

Frequently asked Questions

After clicking on a Portfolio item it doesn't load.

Are you working with Google Chrome? If yes, please note that chrome does not support AJAX when opening the files locally. So please try another browser or upload it to a webserver.

My Logo (or images) are not retina ready, although the @2x version exists.

The retina.js can only apply on webserver, so it doesnt work locally. If you followed the instruction in Retina above, all should be fine once uplaoded to a web server.
